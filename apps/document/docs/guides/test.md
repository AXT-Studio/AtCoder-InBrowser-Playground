# 2. テストする

コードを書き終わったら、書いたコードが正しく動作するかテストしてみましょう！
AIBPは、ブラウザ内でコードを実行することでジャッジをエミュレートする機能を持っています。

:::warning AtCoderジャッジ環境との差異について
AIBPのコードテスト機能は、技術的な都合で、AtCoderのジャッジと一部異なる挙動を示します。
詳細については、[対応環境](./references/supported.md#対応環境)の各言語のセクションを参照してください。
:::
:::warning インタラクティブな問題について
AIBPのコードテスト機能は、「インタラクティブな問題(あなたの作成したプログラムとジャッジプログラムが入出力を介して対話を行う形式の問題)」に対応していません。ご了承ください。
:::

AIBPのコードテスト機能は、「Test」タブから使用します。

![TestタブのUI](/tab_test.webp)

:::details Testタブの各要素の説明
- **Testcase Input** 欄
    - ここにテストケースの入力を記入します。
- **Expected Output** 欄
    - ここに、Testcase Inputに対して期待される出力を記入します。
- **Run Test** ボタン
    - ここまでに記入したTestcase Inputをコードエディターで書いたコードに入力として与えてコードを実行し、出力をExpected Outputと比較します。
- **Status** 欄
    - Run Testの結果が表示されます。
    - AC, WA, TLE, REなどのステータスが表示されます。
- **Exec. Time** 欄
    - コードの実行にかかった時間が表示されます。
- **Example Auto-Exec.** ボタン群
    - 問題文から入出力例を自動抽出できた場合のみ表示されます。
    - Testcase InputとExpected Outputに当該入欲例をコピーし、自動でテストを実行します。
- **Actual Stdout** 欄
    - コードの実行によって実際に出力された標準出力が表示されます。
- **Actual Stderr** 欄
    - コードの実行によって実際に出力された標準エラーが表示されます。
:::

## テストケースを手動入力してテスト

- 「Testcase Input」欄に、テストケースの入力を書きます。
- 「Expected Output」欄に、Testcase Inputに対して期待される出力を書きます。
- 「Run Test」ボタンを押すと、コードテストが開始されます。
- コードテストが完了すると、以下の欄に結果が表示されます。
    - Status …… AC, WA, TLE, CE, RE等のステータス
    - Exec. Time …… コードの実行にかかった時間
    - Actual Stdout …… コードにテストケースを与えて実行したときの、実際に出力された標準出力
    - Actual Stderr …… コードにテストケースを与えて実行したときの、実際に出力された標準エラー

:::tip 初回実行時の遅延について
各言語の初回実行時などに、コード実行の前に実行環境の初期化処理が行われることがあります。
その場合、実行開始までに数秒かかることがあります。
なお、初期化処理にかかった時間は「Exec. Time」にはカウントされていません。
:::

## 入出力例を自動入力してテスト

「Example Auto-Exec.」欄のボタン群を押すと、以下を行います。

- 問題文中の入力例を取得して「Testcase Input」欄に自動入力
- 問題文中の出力例を取得して「Expected Output」欄に自動入力
- コードテストを開始

## テストの条件設定

一部の問題では、Settingsタブにある以下の設定が役立つことがあります。

![SettingsタブのUI](/tab_settings.webp)

### Test Time Limit (ms)

コードテストの実行時間制限を、ミリ秒単位で設定することができます。
コードテストの実行時間がここで設定された時間を超えた場合、コードの実行完了を待たずに「TLE」として実行を終了します。
実行時間を一時的に伸ばしたい場合などに活用できます。

:::tip 実行時間制限の自動入力
この設定項目は、問題ページを開いたときとページをリロードしたときに、自動的に問題の制限時間に設定されます。
なお、問題文からの制限時間の抽出に失敗した場合、初期値は`2000`msになります。
:::

### Allowable Error

コードテストの想定される出力(Expected Output)と実際の出力(Actual Stdout)の比較において、数値の誤差をどの程度許容するかを設定します。
デフォルト値は`1e-6`(10の-6乗)です。

整数を出力する問題では変更の必要はないでしょう。
小数を出力する問題において、真の値との許容誤差が指示されているときは、この設定項目をその指示に合わせてください。

:::warning
この設定項目は、(Test Time Limitと異なり)問題文からの自動入力が行われません。
:::
:::info
この設定項目の入力欄では、JavaScriptの数値リテラルを受け付けます。
「10の-9乗」を指定したい場合は`1e-9`のように書くと良いでしょう。
:::
