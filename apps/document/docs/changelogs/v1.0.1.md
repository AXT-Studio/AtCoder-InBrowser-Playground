バグ修正と軽微な改善を行った、ver1.0.1をリリースしました。

## 変更点

### fix: JavaScriptのコードテストが実行できない問題([#49](https://github.com/AXT-Studio/AtCoder-InBrowser-Playground/issues/49))を修正

JavaScriptのコードテストを実行しようとすると、必ず「CE」となり実行に失敗する問題を修正しました。

### feat: JavaScript・TypeScriptにおいて、テスト実行可能なコードの条件を緩和([#50](https://github.com/AXT-Studio/AtCoder-InBrowser-Playground/issues/50))

JavaScript・TypeScriptのコードテスト実行において、標準入力の受け渡しに関する処理を一部変更しました。
これにより、JavaScript・TypeScriptのコードテストにおいて、テスト実行可能なコードの条件が緩和されました。

- 旧条件: 以下のすべてを満たすこと
    1. 標準入力を、以下のいずれかのコードのうちいずれかで受け取っていること
        - `require("fs").readFileSync("/dev/stdin", "utf8")`
        - `await Deno.readTextFile("/dev/stdin")`
        - `await Bun.file("/dev/stdin").text()`
    2. 上記のコードで受け取った標準入力を、直接`Main`関数の引数として渡していること
- 新条件: 以下のすべてを満たすこと
    1. 標準入力を、以下のいずれかのコードのうちいずれかで受け取っていること
        - `require("fs").readFileSync("/dev/stdin", "utf8")`
        - `await Deno.readTextFile("/dev/stdin")`
        - `await Bun.file("/dev/stdin").text()`
    2. *(廃止)*

つまり、これまで(〜ver1.0.0)では、以下のようなコードしかテスト実行ができませんでした。

::: code-group
```js [Node.js]
function Main(input) {
    // ここにコードを記述
}
Main(require("fs").readFileSync("/dev/stdin", "utf8"));
```
```js [Deno]
function Main(input) {
    // ここにコードを記述
}
Main(await Deno.readTextFile("/dev/stdin"));
```
```js [Bun]
function Main(input) {
    // ここにコードを記述
}
Main(await Bun.file("/dev/stdin").text());
:::

ver1.0.1以降では、例えば以下のようなコードもテスト実行ができるようになりました。

::: code-group
```js [Node.js]
const input = require("fs").readFileSync("/dev/stdin", "utf8");
// ここにコードを記述
```
```js [Deno]
const input = await Deno.readTextFile("/dev/stdin");
// ここにコードを記述
```
```js [Bun]
const input = await Bun.file("/dev/stdin").text();
// ここにコードを記述
```
:::
