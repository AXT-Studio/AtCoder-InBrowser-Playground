## 変更点

### fix?: JS・TSで`console.log()`を使ったときの挙動を出力を変更 (Discussion [#57](https://github.com/AXT-Studio/AtCoder-InBrowser-Playground/discussions/57))

とくにオブジェクトを`console.log()`などで出力したときに、デバッグがしやすいような形で出力されるようにシリアライズ方法の変更を行いました。
この影響で、JavaScriptおよびTypeScriptのテスト実行時間がわずかに伸びた可能性があります。

以下に主な出力の変化を示します。

| `console.log()`する値 | 〜ver1.1.0 | ver1.1.1〜 | 参考: Node.js |
|:----:|:----:|:----:|:----:|
| `-0` | `0` | `-0` | `-0` |
| `[NaN]` | `[ null ]` | `[ NaN ]` | `[ NaN ]` |
| `[Infinity]` | `[ null ]` | `[ Infinity ]` | `[ Infinity ]` |
| `new Set([1, 2, 3])` | `{}` | `Set (3) {1, 2, 3}` | `Set(3) { 1, 2, 3 }` |

(Symbolを除く)プリミティブな値を単に`console.log()`した場合は、Node.js・Deno・Bunと同じ出力になるように調整を行っています。
オブジェクトを出力する場合はNode.js・Deno・Bunと異なる出力をする場合があります。ただし、デバッグに有用な出力を行うようになっています。

### fix: 一度コードテストがTLEしたあと、次のコード実行が受け付けられない不具合を修正 ([#61](https://github.com/AXT-Studio/AtCoder-InBrowser-Playground/issues/61))

コードテストがTLEとなったときに実行が打ち切られず、結果として内部で次のコード実行を開始することができなくなっていた不具合を修正しました。
