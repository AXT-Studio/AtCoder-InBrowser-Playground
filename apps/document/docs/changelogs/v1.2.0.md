## 変更点

### fix: マウスホバー時のツールチップ表示が正しくJSDocを解釈できず表示が崩れる場合がある問題の暫定修正 ([#63](https://github.com/AXT-Studio/AtCoder-InBrowser-Playground/issues/63))

特定の条件を満たすJSDocをもとに表示されるツールチップの表示が崩れる問題について、暫定修正を行いました。

例えば以下のようなコードについて、`double`にマウスホバーしたり`double(`を入力した場合のツールチップが正しく表示されるようになります。

```ts
/**
 * Double `n`.
 *
 * @example
 * ```ts
 * console.log(double(2)); // 4
 * ```
 *
 * @param n - The number you want to double
 * @returns a
 */
const double = (n: number): number => n * 2;
```

:::details 原因についての補足
この問題は、Monaco Editor自体のバグによるものです。
Monaco Editorに修正PR([#5350](https://github.com/microsoft/monaco-editor/pull/5350))を提出しているため、これがマージされればAIBP以外でも同様の修正が適用されます。
今回のバグ修正は、`pnpm patch`により同様の修正を先行して適用したものです。
:::
