## 変更点

### feat: `InputScanner`を使うパターンのTypeScriptテンプレを追加([#59](https://github.com/AXT-Studio/AtCoder-InBrowser-Playground/issues/59))

`InputScanner`を含むTypeScriptのテンプレートを追加しました。

- `const scanner = new InputScanner(inputText);`の形で`InputScanner`インスタンスが作成されています。
- `InputScanner`は、空白文字(改行含む)を区別せずに、空白文字で区切られたそれぞれを1つのトークンとみなして、トークン列を順に読み出します。
    - Rust環境の`proconio`と似た扱いをしています。
- `scanner.str()`, `scanner.num()`, `scanner.int()`, `scanner.bigint()`で、次のトークンをそれぞれ文字列・数値・整数(number)・整数(bigint)として読み出します。
    - トークンを消費しきったときに`undefined`を返す挙動を踏まえて戻り値の型が`T | undefined`になっているので、(問題ないことを確認したうえで)`scanner.str()!`のようにアサーションをつけてください。
- 複数のトークンを連続で(まとめて)読み出したい場合、これらの関数の引数に読み出したい個数を渡すことができます。
    - 引数なしであればトークンをそのまま、引数ありだとトークンの配列が戻ります。
